HTML Tags Documentation

Comprehensive reference for all HTML tags used in the Personal Debt Tracker, with special focus on form elements.

Form Elements

<form>

Container for all form controls used to add new debts.

Attribute Value Purpose
id "debt-form" Unique identifier for JavaScript to reference the form
method (implied) Defaults to GET, but handled by JavaScript preventDefault()
action (none) Form submission is handled by JavaScript event listener

Form Structure

The form contains four input groups wrapped in <div class="form-group"> containers:

  1. Name input (text)
  2. Amount input (number)
  3. Type selection (dropdown)
  4. Date input (date picker)

Plus two buttons: Clear (reset) and Add Debt (submit).

<input>

Creates various form control fields.

Type Attributes Purpose
Text
type="text"		 id="name"
required 		For entering the person's name. Required field.
Number
type="number"		 id="amount"
step="1"
min="0"
required 		For debt amount. Only accepts positive numbers.
step="1" restricts to whole numbers (no decimals)
Date
type="date"		 id="date"
required 		For selecting the debt date. Defaults to current date.

<select>

Creates a dropdown menu for debt type selection.

Attribute Value Purpose
id "type" Identifies the dropdown in JavaScript
required (boolean) User must select an option

Dropdown Options

The <select> contains three <option> elements:

  • Default placeholder: <option value="">Select...</option>
  • They owe me: <option value="owed-to-me">They owe me</option>
  • I owe them: <option value="i-owe">I owe them</option>

The value attributes are used in JavaScript to determine debt type.

<button>

Interactive buttons for form actions.

Button Attributes Purpose
Clear
type="reset"		 class="btn-clr" Resets all form fields to their default values
Add Debt
type="submit"		 (none) Triggers the form submission handled by JavaScript

Structural Elements

<table>

Displays debts in a tabular format with two tables:

  • id="owed-to-me-table" - For debts owed to the user
  • id="i-owe-table" - For debts the user owes

Table Structure

Each table contains:

<thead>
  <tr>
    <th>Name</th>
    <th>Amount</th>
    <th>Date</th>
    <th>Actions</th>
  </tr>
</thead>
<tbody>
  <!-- Dynamically populated by JavaScript -->
</tbody>

<div>

Container elements used throughout the application:

Class Purpose
header Top section containing logo and title
dashboard Container for the two summary cards
card Styled container for each summary (owed to me/I owe)
form-container Wrapper for the entire debt entry form
form-group Groups each label/input pair in the form

<script>

Links to external JavaScript file that provides all functionality:

<script src="script.js"></script>

Form-Related JavaScript

The script handles:

  • Form submission event listening
  • Input validation
  • Debt object creation
  • Adding to the debts array
  • Updating the UI
  • Form reset functionality
